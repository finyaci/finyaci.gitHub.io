计算机网络系统分层结构的优点

  1. 分层结构使得各层之间是独立的。下层给上层提供服务，上层只需要调用下层提供的接口，所以它降低了问题的规模和复杂度。
  2. 分层结构具有良好的可扩展性。因为降低层间耦合性降低了，修改某个层时，只要不涉及层间接口，就不会对其他层造成影响。同时还有效避免了木桶效应。
  3. 分层架构易于实现和维护。由于整个的系统已被分解为若干个相对独立的子系统，所以在进行调试和维护时，可以对每一层进行单独的调试。

冲突域是基于第一层（物理层），而广播域是机于第二层（数据链路层）

TCP与UDP的区别

  1. TCP面向连接，可靠传输，有重传确认机制 UDP无连接，不可靠，无重传确认机制
  2. TCP面向字节流（把应用层数据看作无结构的字节流），UDP面向报文（对于应用层的报文，既不拆分也不合并，加上UDP首部，一次发送一个报文）
  3. TCP只支持一对一，面向连接的 UDP支持一对一，一对多，多对一，多对多
  4. TCP首部至少20字节 UDP只要8个字节
  5. TCP有拥塞控制和流量控制 UDP没有

1

c++支持java不支持

  1. 缺省函数参数：Java不支持缺省函数参数，而c++支持

  2. 操作符重载：Java不支持操作符重载，而c++支持操作符重载的突出特征，在Java中虽然类大体上可以实现这样的功能，但操作符重载的方便性仍然丢失了不少。Java语言不支持操作符重载是为了保持Java语言尽可能简单。

  3. goto语句：Java不提供goto语句，它虽然指定goto作为关键字，但不支持它的使用。C++还保留了goto语句，但它会引起了程序结构的混乱，不易理解。

  4. Java中没有预处理功能，预处理程序虽然可以很方便地达到许多功能，但是站在软件工程的角度上看，对整个软件的维护其实是很不利的。

java支持c++不支持

  1. 垃圾回收机制：c++用析构函数，手动释放内存，java自动回收（java中有析构函数，但我们一般用不到它）

  2. 多线程。Java允许两个或者多个相同的线程并发运行。这是一种在语言级支持的并发机制。C++中没有类似的机制，或者说C++采用的是单线程的体系结构。如果需要并发执行一个C++程序，必须利用操作系统的功能手工启动。虽然这两种方法都可以同时执行两个或者多个线程，但是Java的方法清楚而且便于使用。　

  3. 异常 JAVA中的异常机制用于捕获例外事件，增强系统容错能力，而C++则没有如此方便的机制。

  4. Java有super关键字，指代父类对象，通常被用于调用父类的构造方法或一般方法C++则没有super关键字

  5. Java有package，可以将类组织起来，便于打包和部署，也有利于类的安全。C++没有这个概念，一个类可以被任意类访问

都支持的但有区别

  1. 多重继承：c++支持多重继承，Java不支持多重继承，但允许一个类继承多个接口，实现了c++多重继承的功能。（多继承坏处：拥有相同名字的实例变量，拥有相同名字的方法子类也没有对其重载，将会产生歧义）

  2. 抽象函数和纯虚函数：java中的抽象函数是只有函数声明，没有方法体，JDK8之后的可以有方法体，非静态方法必须要用default修饰。而c++中的纯虚函数是可以有方法体，在c++中，子类还可以调用父类的纯虚函数

  3. 两者都有this，指代当前对象

速度

  1. java运行在虚拟机上，而c++都是直接编译成可执行文件，是否能跨平台主要看你用到的编译器特性是否有多平台支持。

  2. 因为c和c++是直接编译成可执行文件，所以运行效率要比java高。

存取控制

自主存取控制：用户对不同的数据对象有不同的存取权限，不同用户对同一对象有不同的权限，可转授用户存取权限。
强制存取控制：每一数据对象标以一定密级，每一用户对应某一级别的许可证，只有具有合法许可证的用户才可以存取某一对象。

两者区别

1）自主存取控制机制仅仅通过对数据的存取权限进行安全控制，为数据本身并无安全标记；强制存取控制机制则对数据本身进行密级标记，无论数据如何复制，标记与数据是一个不可分的整体，只有符合密级标记要求的用户才可以操纵数据，从而提供了更高级别的安全性。
2）强制存取控制的安全性级别更高。
3）DAC的数据存取权限由用户控制，系统无法控制；MAC安全等级由系统控制，不是用户能直接感知或进行控制的。

1）视图能够简化用户的操作 
2）视图使用户能以多种角度看待同一数据
3）视图对数据库的重构提供了一定程度的逻辑独立性-数据库的逻辑模式改变了（将原来的表拆分），使用视图，仍然可以按数据库重构之前的要求查询数据
4）视图能够对机密数据提供安全保护

视图和基本表的区别

虚表：视图是从一个或几个基本表导出的表，它与基本表不同，是一个虚表，
只存放视图的定义：数据库中只存放视图的定义，这些数据存放在原来的基本表中，当基本表中的数据发生变化，从视图中查询出的数据也就随之改变
视图一经定义就可以像基本表一样被查询、删除，也可以在一个视图之上再定义新的视图，但是对视图的更新操作有限制

检查点

优点：

  1. 从检查点处开始搜索日志，相较于从头开始搜索日志，减少搜索日志占用的时间

检查点记录的是：建立检查点时刻所有正在执行的事务以及事务在日志中的地址

登记日志文件时必须遵循两条原则： 
1）登记的次序严格按并发事务执行的时间次序。
2）必须先写日志文件，后写数据库。

操作：

  1. 从重新开始文件中找到最后一个检查点文件的地址，并从日志文件中找到该检查点
  2. 建立UNDO-LIST和REDO-LIST，检查点时刻正在执行的暂时事务加入UNDO—LIST (检查点时刻正在执行的事务加入ACITIVE—LIST，ACITIVE—LIST加入UNDO-LIST)
  3. 从检查点处开始扫描日志，有新开始的的事务加入UNDO-LIST，有提交事务则把该事务从UNDO-LIST的移入REDO-LIST
  4. 根据UNDO-LIST和REDO-LIST执行UNDO和REDO操作

死锁的预防

一次封锁法：一次性在所有要用到的对象上加锁
缺点：降低系统的并发性，数据库是在变化的，很难事先确定哪些对象要上锁

超时法：设置一个超时时间，事务等待超过一定时间就判定为死锁。（缺点：容易误判）
等待图法：系统每隔一段时间就生成事务等待图，等待图出现了回路则说明出现了死锁

1.1 SelectionSort

  • 算法思想：每次找到元素中的最小值，依次和最值元素应该在的位置交换位置

  • 比较次数没有多少之分

  • 时间复杂度（平均）：O(n^2)
  • 最好情况：数组已经排好序，n个数据，每轮比较1次，比较n-1轮
  • 最差情况：每一次比较都比当前比较值小，n个数据，比较n-1轮，每次n-i次比较
  • 10个数据：9+8+7+…+1
  • (1+n-1)x(n-1)/2≈n^2/2

1.2 InseartionSort

  • 将第一个元素当作有序的，每一次在另一边取出一个数据，插入到有序的数组中，

  • 具体实现：每一轮选取一个元素依次和有序序列中的元素进行比较，由于左边是一个从小到大的有序序列，选取的策略是，如果这一轮的元素比有序序列中的元素小就交换位置继续比较，如果当前元素比有序序列中的元素大，说明该有序序列左侧所有元素都没有这一轮的比较元素大，这时就可以停止比较了

  • 时间复杂度（平均）：O(n^2)

  • 最好情况：n-1次
  • 最差情况：每一次比较都比当前比较值大，n个数据，比较n-1轮，每次i次比较
  • 10个数据：1+2+…+9
  • (1+n-1)x(n-1)/2≈n^2/2
  • 数组部分有序的情况下排序将更节省时间

  • 时间复杂度（平均，最好，最坏）：O(nlgn)

  • 每一次比较2*(logn/log2-1)次（深度-1），n个元素比较n-1轮

  • 主要思想：将待排序的元素构建成堆，一般有大顶堆和小顶堆，堆顶元素是最值。如果要从小到大排序，则构建小顶堆，输出堆顶元素后，剩余元素继续构建堆，得到次小值，反复这个过程，直到排序结束

堆排序是简单选择排序的改进：简单选择排序在一趟排序中仅选出最小关键码，没有把一趟比较结果保存下来，因而记录的比较次数较多。堆排序在选出最小关键码的同时，也找出较小关键码，减少了在后面的选择中的比较次数，从而提高了整个排序的效率。

  • 大顶堆：非叶子节点均不小于其左右孩子节点的值

  • 小顶堆：非叶子节点均不大于其左右孩子节点的值

  • 建堆过程是从最小的非叶子节点开始，下沉元素是从根节点开始

B*树是B+树的变体，在B+树的非根和非叶子结点再增加指向兄弟的指针；B*树定义了非叶子结点关键字个数至少为(2/3)*M，即块的最低使用率为2/3（代替B+树的1/2）；
B+树的分裂：当一个结点满时，分配一个新的结点，并将原结点中1/2的数据复制到新结点，最后在父结点中增加新结点的指针；B+树的分裂只影响原结点和父结点，而不会影响兄弟结点，所以它不需要指向兄弟的指针；
B*树的分裂：当一个结点满时，如果它的下一个兄弟结点未满，那么将一部分数据移到兄弟结点中，再在原结点插入关键字，最后修改父结点中兄弟结点 的关键字（因为兄弟结点的关键字范围改变了）；如果兄弟也满了，则在原结点与兄弟结点之间增加新结点，并各复制1/3的数据到新结点，最后在父结点增加新 结点的指针；
所以，B*树分配新结点的概率比B+树要低，空间使用率更高；

  3. 将这棵新的二叉树放到原来的数组中，再次排序，重复以上步骤

  • 若该节点的邻接节点都被访问过，则退回上一个节点再进行DFS
  • 每次访问当前节点后都访问当前节点的第一个邻接节点
  • 递归过程

3.3 BFS广度优先

  • 访问开始节点，再访问开始节点的所有相邻节点，再以这些相邻节点为队列，依次从队列中取出节点，访问这些节点的相邻节点

最小生成树MST

  • Minimum Cost Spanning Tree
  • 带权无向联通图，选定一颗生成树，使边上权总和最小
  • N个顶点，N-1条边

3.4 Prim算法（普利姆）- 贪心算法

  • 从一个起点开始每次都将与该点连通的权值最小的边加入生成树
  • n顶点的无向带权图，直至找到n-1条边为止

3.5 Kruskal算法（克鲁斯卡尔）

  • 按权值从小到大，依次都将不成环的最小边加入生成树

最短路径算法

Dijkstra

  • 不能求解权值为负数的图
  • O(n^2)

  • n个点要进行n次循环，每一次只获得一个点的最短路径，对于每个点要也要进行遍历找到相邻的点进行比较，邻接矩阵n行n列，一个点的相邻点有n-1个+1和自己相邻的节点

  • Dijkstra求解的问题：一个点到其余所有点的最小距离

  • 到某一个点的最短路径要么是，起始点直接到该点的距离，要么是起始点经某个已求得的最短路径的点再到该点的距离

算法实现

  • 目的：求出起始点到每一个点的最短距离
  • 初始状态设置一个集合，集合存储的是起始点到各相邻点的距离，如果不相邻就用正无穷或者一个很大的数表示；
  • 每次都选取最短的路径（贪心算法的体现），最短路径的选取要比较两种情况，第一种，下一个要求的顶点和起始点直接相连的；第二种，起始点经过那些已经求得最短路径的顶点再到下一顶点的距离之和（这一类也有多种情况，因为那些已经求得最短路径的点不止一个），选取这些情况下面总路径最短的那一个。

  • 重复以上过程，距离集合中起始点到各点的距离会不断减小，直至最后一个点，这就体现了寻找最短路径的过程

  • 设立两个集合，一个是顶点集合（V），另一个是起始点到各顶点的距离集合（D）(如果不连通距离就设为正无穷)

  • 从距离集合中选取最小的距离di移出集合，并在顶点集合中移出对应的顶点vi
  • 之后对距离集合进行更新，对于刚移出来的点，它有相邻的节点，更新这些相邻节点的距离，但不包括已经被移出集合的点。
    一个是起始顶点直接到这些相邻顶点的距离，另一个是起始顶点经过刚移出来的点到相邻顶点的距离，选择其中最小的距离，更新距离集合，同时记录当前顶点的前驱，用于记录路径。

形式上看

  • Dijkstra的大致思想就是，根据初始点，逐次找到离初始点最近的点。
  • 和Prim算法用的思想很像 使用的是贪心算法
  • 以起始点为中心向外层层扩展(广度优先搜索思想)，直到扩展到终点为止

Folyd

  • Folyd求解的问题：各个顶点到其余所有点的最小距离
  • DIS(v1,v2) < DIS(v1,v3)+DIS(v3,v2)
  • O(n^3) 三从循环 中间点 起始点 终点

算法实现

  • ABC三个点，B为中间点 如果AB+BC的距离小于AC的距离那么就选择AC作为最短路径
  • 对图上所有点都进行如上操作，每次操作都更新 前驱表和距离表（使用邻接矩阵）
  • 三重循环之后就能得到最终结果

Dijkstra VS Folyd

  • Dijkstra求解的问题：一个点到其余所有点的最小距离
  • Folyd求解的问题：各个顶点到其余所有点的最小距离
