5.1

It can be divided into seven phases.

First is planning phase,in this phase, project leaders evaluate the terms of the project. This includes calculating costs, creating a timetable, and creating the teams and leadership structure for the project.

Second is requirements analysis phase ,we determine what the application is supposed to do.We usually get requirements from users.

Third is Design and Prototyping phase ,we can design the program according to SRS.Defines the basic problems we may encounter in software development such as architecture, platforms and User Interface

Fourth is Software development phase,we choose the right language to program.This is the actual writing of the program.A small project might be written in few days.

Fifth is Testing phase, we test our progaram ,find bugs and fix them.

Sixth is Deployment phase, which means the application is made available to users.

Seventh is Operations and Maintenance phase, we have to maintain our program and update them when necessary.

SRS: Software Requirement Specification

11.1

  1. First is System requirements analysis phase,we get actual requirements from our users.
  2. Second is Conceptual structure design phase,we establish conceptual model according to users’ requirements.In RDBMS we usually create Entity Relationship Diagram.
  3. Third is Logical structure design phase,we convert conceptual model to logical model.In RDBMS,we convert ER Diagram to Relationship model.
  4. Fourth is Physical structure design phase,we determine the database access method and storage structure.
  5. Fifth is Database implementation phase,we build database and do trial run.
  6. The last is Database operation and maintenance phase,we need to maintain our database when they in use.And we have to update our database if necessary.
