  2. 方法
    • 与弹出窗口有关的方法

DOM

  • Document Object Model
  • 将标记语言文档各个组成部分封装为对象。使用这些对象，对文档进行CRUD操作

W3C DOM 标准被分为 3 个不同的部分：

  1. 核心 DOM - 针对任何结构化文档的标准模型

    • Document - 文档对象
    • Element - 元素对象
    • Attribute - 属性对象
    • Text - 文本对象

    • Comment - 注释对象

    • Node - 结点对象，以上五个对象的父对象

  2. XML DOM - 针对 XML 文档的标准模型
  3. HTML DOM - 针对 HTML 文档的标准模型

* GenericServlet(extends from Servlet)
    * To write a generic servlet, you need only override the abstract service method.
* HttpServlet(extends from GenericServlet)
    * Provides an abstract class to be subclassed to create an HTTP servlet suitable for a Web site. A subclass of HttpServlet must override at least one method, usually one of these:
        * doGet, if the servlet supports HTTP GET requests
        * doPost, for HTTP POST requests
        * doPut, for HTTP PUT requests
        * doDelete, for HTTP DELETE requests
        * init and destroy, to manage resources that are held for the life of the servlet
        * getServletInfo, which the servlet uses to provide information about itself

1 多态

  • A:虚拟方法调用

    • 成员变量(属性)：
      • 编译看左边(父类)，运行看左边(父类)。
    • 成员方法：
      • 编译看左边(父类)，运行看右边(子类)。
      • 但是子类的特有方法无法调用（向下强转再调用），只能调用父类中定义过的方法，即只能调用父类中定义过的子类方法。
    • 静态方法：
      • 编译看左边(父类)，运行看左边(父类)。
      • (静态和类相关，算不上重写，所以，访问还是左边的)
    • 总结：
      • 只有非静态的成员方法,编译看左边,运行看右边

  • B: 实例：

    • Dog和Cat共同继承Animal类，Animal类中定义一个接收Animal对象的方法，
    • 当传入对象为Dog的实例时，animal.sleep()执行的是Dog实例的方法，而当传入对象为Cat的实例时，animal.sleep()执行的是Cat实例的方法
    • 对于子类特有对象则需要分别转换类型
  • C:
    多态是运行时行为，即编译期无法确定函数调用，只有在运行时才能确定

3.2 泛型通配符

  • 泛型通配符的含义是传递参数是选取符合条件类型中的一种
  • <?> 任意类型中的一种
    • 无限定通配符<?>很少使用，可以用替换，同时它是所有类型的超类
    • 无法使用add
  • <? extends E> 上界通配符
    • 为该实参E或其子类中的一种
    • 使用<? extend E>通配符作为方法参数，表示方法内部代码对于参数可以读，不能写。
    • 对于List，无法进行add操作:
      • 因为无法确定该泛型的实际参数，泛型下限也无法确定；假设B继承E，A继承B，若该泛型参数为子类类型A，根据规则可向其中存放不高于E类型的对象，往其中存放父类类型B，则明显不合适
  • <? super E> 下界通配符
    • 为该实参E或其父类中的一种
    • 使用<? super E>通配符作为方法参数，表示方法内部代码对于参数只能写，不能读。
    • 对于List，可以进行add操作:
      • 可以存放任何E(包含E)的子类对象，因为该泛型的下限确定为E，因而任何E的子类均可存放
  • 对比super和extend
    • <? extends T>允许调用读方法T get()获取T的引用，但不允许调用写方法set(T)传入T的引用（传入null除外）；
    • <? super T>允许调用写方法set(T)传入T的引用，但不允许调用读方法T get()获取T的引用（获取Object除外）。 - get()方法返回的是Object类型
  • 存取原则PECS
    • 如果你想从一个数据类型（Producer）里获取数据，使用 ? extends 通配符
    • 如果你想把对象写入一个数据结构里（Consumer），使用 ? super 通配符
    • 如果你既想存，又想取，那就别用通配符。

  • 和<?>

    • 是参数类型，常常用于泛型类或泛型方法的定义；
    • <?>是通配符，一般不能直接用来定义类或泛型方法，因为它不能直接参与操作，常常用于泛型方法的调用代码或泛型方法的型参。
